# 群晖 NAS 部署指南

## 方法一：使用 Docker（推荐）

### 1. 在群晖上安装 Docker
- 打开套件中心
- 搜索 "Docker"
- 安装 Docker 套件

### 2. 上传项目文件
- 使用 File Station 上传整个项目到 `/docker/nas-material-manager/`
- 或使用 SSH/SCP 上传

### 3. 使用 docker-compose 启动
```bash
# SSH 登录群晖
cd /docker/nas-material-manager
docker-compose up -d
```

### 4. 访问系统
- 浏览器打开: `http://群晖IP:3000`

---

## 方法二：直接运行 Node.js

### 1. 启用 SSH
- 控制面板 → 终端机和 SNMP
- 勾选 "启动 SSH 功能"

### 2. 上传项目文件
- 使用 File Station 上传到 `/homes/你的用户名/nas-material-manager/`

### 3. SSH 登录并安装
```bash
# SSH 登录群晖
ssh admin@群晖IP

# 进入项目目录
cd ~/nas-material-manager

# 设置脚本权限
chmod +x deploy/linux/*.sh

# 运行安装脚本
./deploy/linux/install.sh

# 启动系统
./deploy/linux/start.sh
```

### 4. 设置开机自启（可选）

创建任务计划：
- 控制面板 → 任务计划
- 新增 → 触发的任务 → 用户定义的脚本
- 任务名称：启动 NAS 素材管理系统
- 用户账号：选择你的用户
- 事件：开机
- 任务设置 → 运行命令：
  ```bash
  cd /var/services/homes/你的用户名/nas-material-manager
  nohup ./deploy/linux/start.sh > system.log 2>&1 &
  ```

## 数据持久化

数据存储在项目的 `data/` 目录：
- `data/db/` - 数据库文件
- `data/uploads/` - 上传的素材
- `data/backups/` - 自动备份

**建议定期备份整个 `data/` 目录！**

---

## 端口配置

默认使用端口 3001，如需修改：
- 编辑 `backend/.env`
- 修改 `PORT=3001` 为你想要的端口

## 常见问题

### Q: 如何查看日志？
```bash
# 查看系统日志
tail -f system.log
```

### Q: 如何更新系统？
1. 备份 `data/` 目录
2. 上传新的代码文件（保留 data/ 目录）
3. 重启系统
