# 打包完整离线包指南

## 前置条件

- 已安装 Node.js 18 LTS
- 已安装 git（可选）
- 有网络连接（用于首次下载依赖）

## 步骤

### 1. 安装项目依赖

在项目根目录运行：

```bash
cd backend
npm install
```

这会在 `backend/node_modules` 生成所有依赖包。

### 2. 验证安装

```bash
# 测试启动一下
node src/server.js
```

确认能正常启动后按 Ctrl+C 停止。

### 3. 清理临时文件

```bash
# 删除临时上传文件
rm -rf backend/data/temp/*

# 删除旧日志
rm -f system.log

# 保留空的 data 目录结构
mkdir -p backend/data/db backend/data/uploads backend/data/backups backend/data/temp
```

### 4. 打包

#### Windows 系统
```powershell
# 使用 PowerShell 打包
Compress-Archive -Path * -DestinationPath nas-material-manager-v3-offline.zip
```

#### Linux/Mac 系统
```bash
# 使用 tar 打包
tar -czf nas-material-manager-v3-offline.tar.gz --exclude='.git' --exclude='node_modules/.cache' --exclude='*.log' .
```

### 5. 验证离线包

解压到另一个目录，测试：
1. 不联网是否能正常安装
2. 是否能正常启动
3. 所有功能是否正常

## 离线包内容清单

完整的离线包应包含：

```
nas-material-manager-v3/
├── deploy/                    # 部署脚本和文档（我们刚创建的）
├── backend/
│   ├── src/                  # 后端代码
│   ├── node_modules/         # ❗️这个很重要，所有依赖都在这里
│   ├── package.json
│   ├── package-lock.json
│   ├── .env.example
│   └── data/                 # 空的数据目录
│       ├── db/
│       ├── uploads/
│       ├── backups/
│       └── temp/
├── frontend/                  # 前端代码
├── Dockerfile
├── docker-compose.yml
└── README.md
```

---

## 注意事项

### ⚠️ node_modules 必须包含！
没有 node_modules 就不是离线包，用户部署时还得联网下载依赖。

### ⚠️ 不要包含敏感信息
- 确认 `.env` 没有包含真实的 API Key
- 确认 `data/db/` 没有包含测试数据（除非想做演示包）
- 确认没有包含 `system.log` 等日志文件

### ⚠️ 跨平台注意
- node_modules 是平台相关的！
- 如果在 Windows 打包，只能在 Windows 用
- 如果在 Linux 打包，只能在 Linux 用
- 如果要给多个平台用，需要分别打包

---

## 分平台打包建议

### 做一个 Windows 离线包
1. 在 Windows 机器上操作
2. 运行 `npm install`
3. 打包为 `nas-material-manager-v3-windows.zip`

### 做一个 Linux 离线包
1. 在 Linux 机器（或 WSL）上操作
2. 运行 `npm install`
3. 打包为 `nas-material-manager-v3-linux.tar.gz`

## 快速打包脚本（可选）

在项目根目录创建 `package-offline.sh`（Linux/Mac）：

```bash
#!/bin/bash
cd backend
npm install
cd ..
tar -czf nas-material-manager-v3-$(date +%Y%m%d).tar.gz \
    --exclude='.git' \
    --exclude='backend/node_modules/.cache' \
    --exclude='*.log' \
    --exclude='backend/data/db/*' \
    --exclude='backend/data/uploads/*' \
    --exclude='backend/data/backups/*' \
    --exclude='backend/data/temp/*' \
    .
```

创建 `package-offline.bat`（Windows）：

```batch
@echo off
cd backend
call npm install
cd ..
echo 请手动使用压缩工具打包，确保包含 backend/node_modules
pause
```
